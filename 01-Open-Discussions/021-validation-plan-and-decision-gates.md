# Discussion 021 — Validation Plan and Decision Gates

## Status
Open for GPT × Claude review.

## Scope
Define how the first pilot evaluates AI-assisted creation and AI-assisted Reconcile using predeclared descriptive thresholds without overstating causality.

## Out of Scope
- product behavior redesign
- provider/runtime architecture
- implementation milestone details

## Formal Hypotheses
### H1 — AI-Assisted Creation
Users can reach a useful, credible first plan through bounded AI assistance.

### H2 — AI-Assisted Reconcile
AI can reduce unresolved work into fewer understandable, user-approved decisions.

## Questions
1. What metrics define success, weak success, failure, and inconclusive results for H1?
2. What metrics define the same outcomes for H2?
3. What sample size and exposure count are minimally interpretable?
4. Which metrics are behavioral, self-reported, or operational?
5. How are AI suggestions counted when edited before acceptance?
6. How is plan usefulness distinguished from easy-but-trivial plans?
7. How are Reconcile results segmented by Light, Medium, and Recovery severity?
8. What minimum exposure is required before judging a severity band?
9. Which comparisons are descriptive only, including manual vs AI-assisted source?
10. Which causal claims are explicitly forbidden?
11. What decision follows each result combination across H1 and H2?
12. What evidence requires reducing scope, redesigning UX, or revisiting the thesis?
13. What qualitative interviews or observation accompany event data?
14. How are thresholds locked before pilot data is reviewed?

- decision matrix
