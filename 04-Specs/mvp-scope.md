# MVP Scope Boundary

## Purpose

Define what the first product must prove and what must stay out of scope.

The MVP should not try to become a full life operating system immediately. That would be the usual human tradition of turning a sharp idea into a bloated furniture store.

## Core MVP Bet

Users who abandon planners after unfinished tasks may return more often if the product helps them reconcile slippage without shame.

## MVP Includes — Product Map

These can exist in the broader MVP product map:

- Today page
- Goal creation
- Task creation
- Goal-linked tasks
- Planned-for-day tasks
- Reconcile-on-open flow
- Skippable reconcile
- Done / Carry / Drop actions
- Carry-forward behavior
- Optional task shrinking or clarification
- Append-only event log
- Daily rollup
- Weekly review
- AI goal intake
- AI-assisted goal-to-task breakdown
- User approval before applying AI suggestions
- Basic metrics tracking

## Validation Phase 1 Includes

Phase 1 tests the reconcile hypothesis cleanly.

Enabled:

- manual goal/task entry
- Today page
- planned-for-day tasks
- reconcile-on-open
- skippable reconcile
- Done / Carry / Drop
- weekly review
- event log
- metrics

Disabled or hidden from success measurement:

- AI-generated 7-day planning
- strong AI personalization suggestions
- routines
- time-blocking
- calendar integration

## Validation Phase 2 Includes

Phase 2 starts only after Phase 1 passes or weak-passes.

Enabled:

- AI goal intake
- AI-generated task plan draft
- user approval/edit/reject
- compare manual vs ai_generated task behavior

Required:

- source tagging: `manual`, `ai_generated`, `system`

## MVP Excludes

- Full LMS
- Personality diagnosis
- Talent discovery
- Social/team planning
- Long-term child education system
- Complex ML personalization
- Fully automated planning
- Streak-heavy gamification
- Shame-based overdue UI
- Calendar clone behavior
- Routine system in Phase 1
- Time-blocking in Phase 1

## Later

- Adaptive difficulty
- Energy tagging
- Calendar integration
- File/commit/activity signals
- Project unlocks
- Team/community modes
- LMS integration
- Talent and career path discovery
- Routines
- Time-blocking
- Deadline vs do-date distinction

## First test segment

Self-directed knowledge workers who already tried and abandoned tools like Notion, Todoist, TickTick, Google Calendar, or paper planning.

Initial likely testers:

- developers
- freelancers
- students
- solo builders
